# avito-test-task

Что сделал помимо ТЗ

В API не описано, как обрабатывать отсутствие обязательных параметров. Во всех хендлерах я добавил простую валидацию: если обязательное поле пустое возвращаю 400. 

Также во всех ручках добавлен дефолтный ответ 500 на внутренние ошибки базы.

В Users, в ручке /users/getReview, добавил проверку на существование пользователя. API не описывает этот случай, если пользователь не найден возвращаб 404

В PullRequests, в ручке /pullRequest/reassign

если старый ревьювер не назначен на этот пр, то возвращаю 409

если нет ни одного кандидата для переназначения, то возвращаю 409

Менять саму OpenAPI-спецификацию я не стал, так как не понял можно ли это делать.


##Запуск

Сначала скопировать пример окружения:

cp .env.example .env

Сервис, Postgres и миграции поднимаются одной командой:

```bash
make docker-up
```

после запуска сервис доступен на:

http://localhost:8080
